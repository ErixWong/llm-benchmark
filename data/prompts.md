## 样本分类说明

### tech/ - 技术类文档
专业领域的技术文档，包括分布式系统、人工智能、云计算、数据库等。主要用于测试模型在专业技术领域的理解和生成能力。

**特点**：
- 专业术语密集
- 结构化程度高
- 逻辑性强
- 适合测试专业领域知识

### literature/ - 文学类文档
小说、散文等文学作品的样本，用于测试模型的创意写作和文学理解能力。

**特点**：
- 叙事性强
- 情感和心理描写丰富
- 情节结构复杂
- 适合测试创意生成能力

### news/ - 新闻类文档
新闻报道、新闻汇编等内容，用于测试模型的信息整合和新闻写作能力。

**特点**：
- 信息密度高
- 时效性强
- 客观性强
- 适合测试信息抽取和整合能力

### dialogue/ - 对话类文档
客服对话、医疗问诊、教育辅导等场景的对话数据，用于测试模型的对话理解和生成能力。

**特点**：
- 多轮交互
- 角色切换
- 情境依赖
- 适合测试对话能力

### code/ - 代码类文档
多种编程语言的代码示例，用于测试模型的代码理解和生成能力。

**特点**：
- 多语言覆盖
- 算法实现完整
- 注释详尽
- 适合测试编程能力

### mixed/ - 混合类文档
跨领域的综合内容，包括科学、历史、地理、哲学等多领域知识，用于测试模型的综合知识能力。

**特点**：
- 领域广泛
- 知识密集
- 问答格式
- 适合测试知识储备

## Prompt模板

### 基础总结Prompt

```
请根据以下材料，编写一份结构化的总结报告。要求：

1. **概述**：简要说明材料的主要内容和核心观点
2. **关键要点**：列出3-5个最重要的要点
3. **详细总结**：按章节或主题进行详细总结
4. **结论**：总结材料的价值和启示

请用专业、清晰的语言撰写，保持逻辑性和完整性。
```

### 简短总结

```
请用200字以内的篇幅，总结以下材料的核心内容。
```

### 详细分析

```
请对以下材料进行深入分析：

1. 提取核心论点和论据
2. 分析论证逻辑和方法
3. 评价材料的优缺点
4. 提出改进建议或延伸思考

请确保分析全面、客观、有深度。
```

### 代码理解Prompt

```
请分析以下代码：

1. 解释代码的功能和目的
2. 分析关键算法和数据结构
3. 指出潜在的优化点
4. 给出改进建议
```

### 对话分析Prompt

```
请分析以下对话：

1. 识别参与者的角色和目的
2. 分析对话的结构和流程
3. 评估沟通效果
4. 提出改进建议
```

## 使用示例

### 单文件测试 (约8K tokens)

```bash
node tests/llm-benchmark.test.js -c 4 -r 3 -n 1
# -n 1 表示每次请求随机选择1个样本文件
```

### 多文件组合 (约32K tokens)

```bash
node tests/llm-benchmark.test.js -c 2 -r 2 -n 4
# -n 4 表示每次请求随机选择4个样本文件，组合成约32K tokens的上下文
```

### 大上下文测试 (约64K tokens)

```bash
node tests/llm-benchmark.test.js -c 1 -r 1 -n 8
# -n 8 表示每次请求随机选择8个样本文件
```

## 注意事项

### Token限制

| 模型 | 最大上下文 | 推荐文件数 |
|------|-----------|-----------|
| GPT-3.5-Turbo | 4K / 16K | 1-2个 |
| GPT-4 | 8K | 1个 |
| GPT-4-32k | 32K | 4个 |
| GPT-4-Turbo | 128K | 全部16个 |
| Claude-3 | 200K | 全部16个 |

### 费用提醒

- 大上下文测试会产生较高的API费用
- 建议先用小上下文验证功能
- 128K上下文约消耗126K input tokens

### 响应时间

- 大上下文请求响应时间会显著增加
- 建议设置合理的超时时间
- 默认超时为90秒，可通过环境变量调整

## 环境变量配置

```bash
# .env 文件

API_KEY=your-api-key

```
